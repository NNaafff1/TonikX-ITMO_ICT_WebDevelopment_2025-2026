        # Задание 1 - UDP Сокет

        ## Описание задания

        Реализовать простое клиент-серверное приложение с использованием протокола UDP (User Datagram Protocol). Сервер должен принимать сообщения от клиента и отправлять ответное сообщение.

        ## Теоретические основы

        **UDP (User Datagram Protocol)** - это транспортный протокол, который:
        - Не требует установления соединения
        - Не гарантирует доставку пакетов
        - Не гарантирует порядок доставки
        - Имеет меньшие накладные расходы по сравнению с TCP
        - Подходит для приложений реального времени (стриминг, игры)

        ## Реализация

        ### Серверная часть (server.py)

        ```python
        import socket

        HOST = "127.0.0.1"
        PORT = 8081

        with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as s:
            s.bind((HOST, PORT))
            print(f"UDP server listening on {HOST}:{PORT}")
            while True:
                data, addr = s.recvfrom(4096)
                print(f"Received from {addr}: {data.decode()}")
                s.sendto(b"Hello, client!", addr)
        ```

        **Основные компоненты:**

        1. **Создание сокета**
        - `socket.AF_INET` - использование IPv4
        - `SOCK_DGRAM` - тип сокета для UDP
        
        2. **Привязка к адресу**
        - `bind((HOST, PORT))` - связывает сокет с IP-адресом и портом
        
        3. **Прием данных**
        - `recvfrom(4096)` - принимает до 4096 байт данных
        - Возвращает данные и адрес отправителя
        
        4. **Отправка ответа**
        - `sendto(data, addr)` - отправляет данные на указанный адрес

        ### Клиентская часть (client.py)

        ```python
        import socket

        HOST = "127.0.0.1"
        PORT = 8081

        with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as s:
            s.sendto(b"Hello, server!", (HOST, PORT))
            data, addr = s.recvfrom(4096)
            print("Server replied:", data.decode())
        ```

        **Основные компоненты:**

        1. **Создание UDP сокета**
        - Аналогично серверу, используется `SOCK_DGRAM`
        
        2. **Отправка данных**
        - `sendto(message, (host, port))` - отправляет сообщение на сервер
        - Не требует предварительного установления соединения
        
        3. **Получение ответа**
        - `recvfrom(4096)` - ожидает ответ от сервера

        ## Особенности реализации

        !!! info "Локальный адрес"
            Используется `127.0.0.1` (localhost) для локального тестирования

        !!! note "Размер буфера"
            Буфер размером 4096 байт достаточен для большинства простых сообщений

        !!! warning "Отсутствие гарантий"
            UDP не гарантирует доставку, поэтому пакеты могут теряться

        ## Преимущества и недостатки UDP

        ### Преимущества ✅
        - Низкая задержка
        - Простота реализации
        - Меньшие накладные расходы
        - Не требует установления соединения

        ### Недостатки ❌
        - Нет гарантии доставки
        - Нет контроля порядка пакетов
        - Нет контроля целостности данных
        - Нет управления потоком
